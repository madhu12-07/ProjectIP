Methodology

How the Lens Works

1. HSV Colour Extraction

The uploaded photo is drawn onto a canvas and resized to a fixed working resolution. Every pixel is converted from RGB to HSV (Hue, Saturation, Value). Near-white and near-black pixels are treated as background and down-weighted. The remaining hues are combined using a circular mean — since hue wraps around a colour wheel, a normal average would break down near 0°/360°, so the mean is computed via the average sine and cosine of each hue angle.

2. Sobel Edge Detection

The image is converted to grayscale and convolved with the horizontal and vertical Sobel kernels to estimate the gradient at each pixel. The gradient magnitude highlights petal boundaries, veins, and texture. The fraction of pixels whose gradient exceeds a threshold gives an edge density score — a rough proxy for how frilly, layered, or smooth a bloom's structure is.

3. Nearest-Neighbour Classification

The extracted feature vector — mean hue, mean saturation, mean value, and edge density — is compared against every hand-calibrated reference profile in the collection using a weighted Euclidean distance (hue is weighted most heavily, since it's the most distinctive channel). The closest profile wins, and distances across the whole collection are converted into relative confidence percentages.

A Note on Accuracy

This is a classical, non-machine-learning pipeline built for an Image Processing coursework demonstration. Each reference profile is calibrated from that species' typical colour and structure rather than learned from a large labelled dataset, so results are illustrative of the technique rather than production-grade botanical identification.
